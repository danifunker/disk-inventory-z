# Screenshots

Drop the captured PNGs in this folder using the exact filenames below — the
project README references them directly.

## Shot list

| Filename | What to capture |
| --- | --- |
| `treemap-main.png` | **Hero shot.** The main window right after scanning a volume with varied content — the colored treemap filling most of the window, the file outline on top, and the Kind list visible. This is the first thing visitors see, so make it the most representative. |
| `kind-statistics.png` | The **Kind** statistics table with its color swatches, sorted by size, so the per-file-type breakdown and the color-matching are clear. Can be the kinds pane on its own or a focused crop of the main window. |
| `disk-usage-pie.png` | The **Disk Usage** pie panel from a whole-volume scan (Scanned / Other / Free), including the legend underneath. |
| `scan-settings.png` | The **disk selection** screen with the **Scan Settings** popover open, showing the scan options (Show Package Contents, Ignore Creator Code, Show Physical File Size). |
| `treemap-zoom.png` | A **drilled-in** treemap — zoom into a large folder, or select a single big file so its block is highlighted, to show the explore/zoom behavior. |

Optional extras you could add later: the treemap **settings popover** (the
gear), and the **Full Disk Access** prompt.

## Capture tips

- **Pick a colorful target.** Scan a drive/folder with a real mix of file
  kinds (apps, documents, images, archives) so the treemap shows lots of
  distinct colors rather than one giant block.
- **Capture the window, not the whole screen.** Press `⌘⇧4`, then `Space`,
  then click the window — macOS captures just that window with a clean
  drop shadow. Files save to the Desktop by default.
- **Be consistent.** Use one appearance (all light mode *or* all dark mode)
  and a similar window size across every shot.
- **Mind privacy.** A fresh VM is ideal — no real filenames, home-folder
  paths, account names, or serial numbers in frame.
- **Resolution.** Retina captures are fine; GitHub scales them down. Keep
  the hero around 1600px wide or less so the page stays light.
